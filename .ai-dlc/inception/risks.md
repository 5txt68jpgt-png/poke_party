# Risks（リスク分析）- PokeParty

## リスク評価基準

- **影響度**: High / Medium / Low
- **発生確率**: High / Medium / Low
- **リスクスコア**: 影響度 × 発生確率

---

## 技術的リスク

### RISK-1: PokeAPI の日本語データ品質
| 項目 | 内容 |
|------|------|
| **リスク** | PokeAPIの日本語名が不完全・不正確な場合がある |
| **影響度** | High - 子供に正しく伝わらないと価値がない |
| **発生確率** | Medium - 公式データだが、最新ポケモンは未対応の可能性 |
| **リスクスコア** | High |
| **軽減策** | 1. 事前にAPIの日本語対応状況を調査<br>2. 未対応の場合は補完データを用意<br>3. 最悪の場合、対応世代を絞る |
| **担当** | 開発初期に検証 |

---

### RISK-2: AIが不適切なポケモンを選定
| 項目 | 内容 |
|------|------|
| **リスク** | AIがテーマを正しく解釈せず、期待と違うポケモンを返す |
| **影響度** | Medium - 再生成で対応可能だが、UX低下 |
| **発生確率** | Medium - プロンプト設計に依存 |
| **リスクスコア** | Medium |
| **軽減策** | 1. プロンプトの事前テスト・改善<br>2. PokeAPIデータを活用した検証ロジック<br>3. 「このポケモンは実在しない」を防ぐバリデーション |
| **担当** | AI統合フェーズ |

---

### RISK-3: AIが存在しない技を提案
| 項目 | 内容 |
|------|------|
| **リスク** | AIがそのポケモンが覚えられない技を提案する |
| **影響度** | High - 「その技覚えないよ！」と子供に指摘される |
| **発生確率** | High - LLMのハルシネーション |
| **リスクスコア** | High |
| **軽減策** | 1. AIはポケモン選定のみ担当<br>2. 技はPokeAPIの習得可能技リストから選定<br>3. AI提案を必ずAPIデータで検証 |
| **担当** | アーキテクチャ設計で対応 |

---

### RISK-4: OpenAI API のレイテンシ
| 項目 | 内容 |
|------|------|
| **リスク** | API応答が遅く、10秒以上かかる |
| **影響度** | Medium - UX低下、離脱の原因に |
| **発生確率** | Medium - ネットワーク状況やAPI負荷に依存 |
| **リスクスコア** | Medium |
| **軽減策** | 1. ストリーミングレスポンスで体感速度向上<br>2. タイムアウト設定と適切なエラー表示<br>3. より軽量なモデル（GPT-3.5-turbo）の検討 |
| **担当** | 実装フェーズ |

---

### RISK-5: PokeAPI の可用性
| 項目 | 内容 |
|------|------|
| **リスク** | PokeAPIがダウンしてデータ取得不可 |
| **影響度** | High - サービス全体が利用不可 |
| **発生確率** | Low - 一般的に安定したAPI |
| **リスクスコア** | Medium |
| **軽減策** | 1. 必要なデータをビルド時にキャッシュ<br>2. フォールバック用の静的データ準備<br>3. エラー時の適切なメッセージ |
| **担当** | アーキテクチャ設計 |

---

## 運用リスク

### RISK-6: OpenAI API 利用コスト超過
| 項目 | 内容 |
|------|------|
| **リスク** | 想定以上の利用でAPI費用が高額に |
| **影響度** | Medium - 個人プロジェクトでの継続困難 |
| **発生確率** | Low - 個人利用なら限定的 |
| **リスクスコア** | Low |
| **軽減策** | 1. 月額上限の設定<br>2. レート制限の実装<br>3. 使用量のモニタリング |
| **担当** | 運用フェーズ |

---

### RISK-7: ポケモン関連の著作権・商標
| 項目 | 内容 |
|------|------|
| **リスク** | ポケモンのデータ・画像使用に関する法的問題 |
| **影響度** | High - サービス停止の可能性 |
| **発生確率** | Low - PokeAPIは広く使われている、個人利用 |
| **リスクスコア** | Medium |
| **軽減策** | 1. 個人利用・非商用として運用<br>2. 公式APIデータのみ使用<br>3. 収益化しない |
| **担当** | 運用全般 |

---

## ユーザビリティリスク

### RISK-8: 親がテーマを思いつかない
| 項目 | 内容 |
|------|------|
| **リスク** | ユーザーがどんなテーマを入力すればいいか迷う |
| **影響度** | Medium - 初回利用の障壁 |
| **発生確率** | Medium - ポケモン知識がないため |
| **リスクスコア** | Medium |
| **軽減策** | 1. テーマ例を入力欄に表示<br>2. ワンタップで使えるおすすめテーマボタン<br>3. 空欄でも「ランダムに選んで」として処理 |
| **担当** | UI設計 |

---

## リスクサマリー

| リスクスコア | リスク |
|--------------|--------|
| **High** | RISK-1（日本語データ）, RISK-3（存在しない技） |
| **Medium** | RISK-2（AI選定）, RISK-4（レイテンシ）, RISK-5（API可用性）, RISK-7（著作権）, RISK-8（テーマ迷い） |
| **Low** | RISK-6（コスト超過） |

---

## 優先対応事項

1. **RISK-3**: AI＋PokeAPIの役割分担で解決（アーキテクチャで対応）
2. **RISK-1**: 開発初期にPokeAPIの日本語データ品質を検証
3. **RISK-8**: UIでテーマ例・おすすめを提示
